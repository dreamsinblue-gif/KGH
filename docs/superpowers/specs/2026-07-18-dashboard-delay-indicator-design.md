# 관리자 대시보드 경과시간/지연 표시 — 설계

## 배경 / 목표

`2_관리자대시보드(admin).html`의 오늘 현황 대시보드는 지점별로 상태 점(대기/운송중/완료)과 시작·완료 시각만 보여준다. 관리자가 "이 배송이 평소보다 오래 걸리고 있는지"를 한눈에 알 수 없다. 이번 설계는 대시보드에 **진행중/완료 배송의 경과·소요시간과, 평균 대비 지연 여부**를 표시하는 것을 목표로 한다.

## 결정한 접근 방식

- **판단 기준**: 수도권 지점은 **그 지점 자신의 월평균** 대비, 지방 지점은 **지방 전체(12개 지점) 합산 월평균** 대비로 지연을 판단한다. 지방 지점은 직접 배송이 아니라 중간 허브까지의 소요시간만 기록되므로([3_개발기획서.md](../../../3_개발기획서.md) 참고), 개별 지점 이력만으로는 통계가 부족하고 의미도 약하다 — 지방 전체를 하나의 공정으로 보고 합산 평균을 기준으로 삼는다.
- **지연 임계치**: 기준(평균) 대비 **+15분 이상**일 때 "지연"으로 표시한다. (통계 페이지의 ±5분 증감 색상 표시와는 별개의, 더 관대한 임계치 — 사소한 변동까지 매번 지연으로 표시되는 것을 피하기 위함.)
- **표시 범위**: 운송중(진행중) 항목과 완료된 항목 모두. 진행중은 실시간 경과시간, 완료는 실제 소요시간을 같은 판단 로직으로 처리한다.
- **데이터 로딩**: 기존에는 통계 페이지(`view === "stats"`) 진입 시에만 `subscribeStats()`로 30일치 데이터를 지연 로딩했다. 이 설계는 로그인 성공 직후(대시보드 진입 시점)에 `subscribeStats()`를 함께 호출하도록 앞당긴다 — 대시보드의 지연 판정과 통계 페이지의 월평균이 항상 같은 데이터/계산식을 공유해야 두 화면이 서로 다른 기준으로 어긋나는 일이 없다. (검토했지만 채택하지 않은 대안: 대시보드 전용으로 더 짧은 기간을 별도 집계 — 통계 페이지와 기준이 달라져 지점이 화면마다 다르게 보이는 비일관성이 생기므로 기각.)

## 1. 데이터 로딩 & 판단 로직

- `initFirebase()`의 `onAuthStateChanged` 콜백에서, 로그인 성공 시 `subscribeDashboard()`와 함께 `subscribeStats()`도 즉시 호출한다. 기존 `statsSubscribing` 가드가 있으므로 이후 사용자가 통계 페이지로 이동해도 중복 구독되지 않는다.
- 새 함수 `expectedMinutes(store, region)`:
  - `region === "수도권"` → 해당 지점의 월평균 (`storeStats(store, ...).monthly`, 기존 로직 재사용)
  - `region === "지방"` → 지방 전체 12개 지점을 합산한 월평균 (지점명 필터 없이 `region === "지방"`인 완료 건 전체의 평균)
  - 평균을 계산할 이력이 아예 없으면 `null` 반환
- 새 함수 `isDelayed(entry)`:
  - 진행중: `경과분 = 현재시각 - startAt`
  - 완료: `경과분 = endAt - startAt`
  - `expectedMinutes(entry.store, entry.region)`이 `null`이면 판단 불가(지연 여부 표시 안 함)
  - 아니면 `경과분 - 기준분 >= 15` 일 때만 지연

## 2. 대시보드 UI 표시 방식

- 지점 row(`[상태점, 지점명, 시작시각, 완료시각]`)에 경과/소요 표시 + 지연 뱃지를 추가한다.
  - 진행중: "OO분째" 형식으로 실시간 경과시간 표시. `isDelayed()`가 true면 빨간 "지연" 뱃지 노출.
  - 완료: 실제 소요분(`완료-시작`) 표시. 초과 시 동일한 "지연" 뱃지.
  - 대기(시작 전): 표시 없음.
  - 판단 불가(이력 없음): 뱃지 없이 소요분 숫자만 표시(소요분도 없으면 `-`).
- **실시간 틱**: Firestore `onSnapshot`은 문서 변경 시에만 재렌더링되므로, 진행중 항목의 "OO분째"가 시간 경과만으로 저절로 갱신되지 않는다. 대시보드 진입 시 `setInterval(() => render(), 60000)`을 시작해 1분마다 화면을 다시 그린다 (Firestore 재구독 없이 클라이언트 시각만 다시 계산). 로그아웃/화면 이탈 시 `clearInterval`로 정리한다.

## 3. 지방점 안내문구 & 예외 처리

- 지방 `region-card` 헤더 아래에 안내문구를 추가한다 (index.html의 기존 "*지방점 운송완료는 중간 터미널 도착 기준" 문구와 톤 통일):
  > *지방점 소요시간은 중간 허브 도착 기준이며, 지연 판정은 지방 전체 평균 대비로 계산됩니다
- 이력이 없어 평균을 낼 수 없는 지점(신규 지점 등)은 지연 뱃지 없이 소요분 숫자만 표시하거나, 그마저 없으면 `-`로 표시한다.

## 4. 에러 처리 & 검증 방법

- 이 기능은 이미 로드된 Firestore 데이터를 클라이언트에서 계산만 하므로 새로운 실패 케이스는 없다. 기존 `loadError` 배너 로직을 그대로 유지한다.
- 테스트 스위트가 없으므로 수동으로 검증한다:
  1. 진행중 항목의 "OO분째" 표시가 1분마다 자동 갱신되는지 확인
  2. 지점 평균보다 15분 이상 걸린 진행중/완료 건에 "지연" 뱃지가 뜨는지 확인
  3. 지방 지점은 개별이 아니라 지방 전체 평균 기준으로 판정되는지 확인
  4. 이력이 전혀 없는 지점은 뱃지 없이 표시되는지 확인
  5. 지방 안내문구가 노출되는지 확인
