# 운송기사 배송 입력 시스템 — 과제 보고서

## 과제명

운송기사 배송 입력 시스템 (배송 트래킹 자동화)

## 과제선정 이유

- 운송기사가 지점별 운송시작·운송완료 시각을 모바일에서 직접 입력하고, 관리자가 이를 실시간으로 확인할 수 있는 체계가 필요했음
- 별도 서버 구축 없이 Firebase(실시간 데이터베이스)와 Google Apps Script(자동 백업)만으로 구현이 가능해, 도입·운영 비용과 유지보수 부담이 낮음
- 등록 즉시 Google 스프레드시트로 자동 집계되어, 수기 정리 없이도 소요시간까지 데이터로 축적됨

## 과제 유형

업무 프로세스 자동화 (서버리스 웹앱 + No-code 자동화 연동)

## 결과물(서비스) 상세 설명

### 시스템 구성

| 구성 요소 | 역할 | 비고 |
| --- | --- | --- |
| 기사 앱 (index.html) | 지점 선택 · 운송시작/완료 입력 | Firebase SDK 내장, PC·모바일 반응형 |
| Firestore | 실시간 데이터베이스 (익명 인증) | onSnapshot 기반 push 동기화 |
| Google Apps Script | Firestore → 스프레드시트 자동 백업 | 5분 주기 트리거, 서비스계정 불필요(OAuth) |
| Google 스프레드시트 | 영구 기록 저장 및 리포트 | 소요시간 자동 계산 수식 포함 |
| 관리자 앱 (admin.html) | 실시간 현황 조회 · 소요시간 통계 | 읽기 전용, 이메일/비밀번호 로그인 |

### 주요 기능

**기사 앱**
- 권역(수도권/지방) 선택 → 지점 다중 선택 → 일괄 운송시작 등록 (동시 시작시각 기록)
- 지방 지점은 "전체 선택" 지원, 안내문구로 "지방점 운송완료는 중간 허브 도착 기준"임을 명시
- 동시 기사 중복 등록 방지 — 오늘 이미 등록된 지점은 선택 화면에서 자동 비활성화(등록됨/완료 표시)
- 운송완료는 확인창을 거치며(되돌릴 수 없음을 안내), 완료되면 목록에서 사라짐
- **지방 일괄 운송완료** — 한 트럭으로 묶여 같은 허브에 동시 도착하는 지방 지점들을, 진행중인 지점이 2개 이상이면 한 번에 완료 처리하는 버튼 제공
- 자정을 넘겨 기기를 켜둬도 새로고침 없이 날짜가 자동 갱신됨
- **테스트 모드**(`?test=1`) — 실제 데이터베이스에 연결하지 않고 샘플 데이터로 모든 기능을 미리 확인할 수 있는 격리된 모드

**관리자 앱**
- 이메일/비밀번호 로그인 (Firebase Authentication)
- 36개 지점(수도권 24 · 지방 12) 전체를 대기/운송중/완료 3색으로 실시간 표시
- 지연 판정 배지 — 지점별(수도권) 또는 지방 전체 평균 대비 경과시간이 기준을 초과하면 표시
- 최근 30일 소요시간 통계 — 지점별 월평균과 조회일 소요시간을 비교해 증감 표시

**공통**
- 양쪽 상단에 교보문고 CI(심볼+워드마크) 삽입, 기존 배경색과 자연스럽게 어울리도록 처리
- 별도 서버 없이 Firebase Firestore + Hosting + Apps Script로만 구성된 완전 서버리스 아키텍처
- 소스는 GitHub 저장소로 버전 관리, 여러 대의 컴퓨터에서 이어서 작업 가능

## 서비스 형태

- 설치 없이 브라우저로 접속하는 반응형 웹앱 (PC·모바일 겸용)
- Firebase Hosting을 통한 정적 파일 배포, 실시간 데이터는 Firestore가 push로 동기화
- 백엔드 서버 없음 — Firestore(DB) + Apps Script(배치성 자동화)만으로 운영
