# 운송기사 배송 입력 시스템 개발 기획서

*PROJECT PLAN*

## 1. 개요

운송기사가 지점별 운송시작·운송완료 시각을 모바일/PC에서 직접 입력하고, 관리자가 실시간으로 확인하며 Google 스프레드시트로 자동 백업·집계할 수 있는 배송 트래킹 시스템입니다. 별도 서버 구축 없이 Firebase와 Google Apps Script만으로 실시간 동기화와 데이터 축적을 구현했습니다.

## 2. 시스템 구성

| 구성 요소 | 역할 | 비고 |
| --- | --- | --- |
| index.html (기사 앱) | 지점 선택 · 운송시작/완료 입력 UI | Firebase SDK 내장, 반응형 |
| Firestore | 실시간 데이터베이스 (익명 인증) | onSnapshot으로 push 동기화 |
| Google Apps Script | Firestore → 스프레드시트 자동 백업 | 5분 주기 트리거, 서비스계정 불필요 |
| Google 스프레드시트 | 영구 기록 저장 및 리포트 | 소요시간 자동 계산 수식 포함 |
| admin.html (관리자 앱) | 실시간 현황 조회 · 소요시간 통계 | 읽기 전용, 같은 Firestore 구독 |

## 3. 화면 구성 (구현 완료)

| 화면 | 주요 기능 |
| --- | --- |
| 1단계 · 등록 | 권역(수도권/지방) 라디오 선택 → 지점 카드 다중 선택. 지방 선택 시 "전체 선택" 카드 제공(전체선택/해제 토글) 및 안내문구(지방점 운송완료는 중간 터미널 도착 기준) 표시 |
| 2단계 · 확인 | 선택된 지점을 칩 목록으로 확인 후 "운송시작" 클릭 시 선택 지점 전체를 일괄 등록 (동시 시작시각 기록) |
| 등록 목록 | 운송시작 → 운송완료 아이콘 좌우 흐름 표시. "운송완료"는 확인창을 거치며(되돌릴 수 없음을 안내), 완료되면 목록에서 사라짐. 지방 지점이 2개 이상 진행중이면 "지방 N개 지점 허브 도착(일괄 운송완료)" 버튼이 나타나 트럭 단위로 한 번에 완료 처리 가능 |
| 관리자 대시보드 | 36개 지점(수도권 24 · 지방 12)을 입력 시스템과 동일한 순서로 전체 표시, 대기/운송중/완료 3색 상태 표시, 실시간 반영. 우측 상단 "소요시간 통계" 아이콘으로 통계 페이지 이동 |
| 소요시간 통계 | 조회일(최근 30일) 선택, 지점 순서 고정. 지점별 월평균 소요시간과 조회일 소요시간을 비교해 증감(색상 구분) 표시. 30일치를 1회 구독해 날짜 변경 시 추가 요청 없이 즉시 재계산 |

## 4. 데이터 구조

**Firestore (deliveries 컬렉션)**
- store, region, date, startAt(Timestamp), endAt(Timestamp|null), createdAt(서버 타임스탬프)

**Google 스프레드시트 (배송기록 시트)**
- ID · 날짜 · 권역 · 지점 · 운송시작 · 운송완료 · 소요시간(자동 수식 =F-E)

## 5. 진행 현황

- 기사 입력 UI (2단계 등록 흐름) 완성
- Firestore 실시간 동기화 완성
- 지점 다중 선택 · 지방 전체선택 기능 완성
- Apps Script 자동 백업 · 소요시간 자동계산 완성
- 관리자 대시보드 · 소요시간 통계 페이지 완성
- **실제 Firebase 프로젝트(`delivery-tracker-9b6d7`) 생성 및 배포 완료** — 기사·관리자 앱 모두 Firebase Hosting에 라이브로 올라가 있고, Firestore 보안 규칙도 배포되어 실제 동작 확인됨 (배포 URL은 CLAUDE.md 참고)
- **관리자 로그인(이메일/비밀번호) 실제 적용 완료** — 계정 생성 및 접근 제한 동작 확인
- **Apps Script 자동 백업 실동작 확인** — 5분 주기로 Firestore → 스프레드시트 반영되는 것을 실데이터로 검증
- **동시 기사 중복 등록 방지** — 여러 기사가 동시에 앱을 쓰는 상황을 고려해, 1단계 지점 선택 화면에서 오늘 이미 등록된 지점은 비활성화(등록됨/완료 표시)되어 중복 등록이 원천 차단됨
- **완료 처리 방식 변경** — (검토 끝에 결정) 완료된 항목은 목록에서 바로 사라지는 방식으로 확정. "초기화" 버튼은 없애고, 대신 운송완료 클릭 시 "되돌릴 수 없다"는 확인창을 먼저 거치도록 해서 오조작을 예방
- **지방 지점 일괄 운송완료** — 지방 지점들은 한 트럭으로 묶여 같은 중간 허브에 동시 도착하므로, 시작할 때처럼("전체 선택") 완료할 때도 진행중인 지방 지점을 한 번에 완료 처리하는 버튼 추가
- **자정 자동 갱신** — 기기를 자정 넘게 켜둬도 새로고침 없이 날짜가 자동으로 갱신되도록 개선
- **교보문고 CI 로고 삽입** — 기사·관리자 앱 상단에 회사 CI(심볼+워드마크) 삽입, 기존 배경색과 자연스럽게 어울리도록 처리
- **테스트 모드 추가** — 기사 앱 URL에 `?test=1`을 붙이면 실제 데이터베이스에 연결하지 않고 샘플 데이터로만 동작하는 테스트 전용 화면 제공 (실서비스 데이터에 영향 없이 기능을 미리 확인 가능)
- **GitHub 저장소 연동** — https://github.com/dreamsinblue-gif/KGH 에 소스 백업, 여러 대의 컴퓨터에서 이어서 작업 가능
